# Kafka란?
- 대규모 데이터 처리에 적합한 **분산형 이벤트 스트리밍 플랫폼**
- 지속적으로 발생하는 로그·이벤트 데이터를 **안정적으로 수집, 저장, 전달**
- 메시지를 단순 전달하지 않고 **시간 순서대로 영속 저장(Event Log)**

**주요 역할**
- 사용자 행동, 서버 로그 등 이벤트 수집(Ingestion)
- Producer–Consumer 분리로 확장성 확보
- 여러 시스템이 동일 데이터를 각자 속도로 소비 가능

**사용 시점**
- 실시간 분석, 스트리밍 처리, 향후 확장 고려 시
- 이벤트 기반 아키텍처의 중심 역할

---

# Flink란?
- **실시간 스트림 처리 프레임워크**
- 이벤트가 발생하는 즉시 데이터를 처리하고 상태를 유지

**주요 특징**
- 저지연 실시간 처리
- 사용자별 상태(State) 관리
- Window 기반 집계, 패턴 탐지 가능

**일반 흐름**
- Data Source → Kafka → Flink → DB/Cache

---

# Spark란?
- **대규모 데이터 배치 처리 프레임워크**
- 주기적으로 쌓인 데이터를 빠르게 분석·집계

**주요 특징**
- 메모리 기반 분산 처리
- SQL, DataFrame, ML 등 다양한 분석 지원
- 실시간보다는 배치 처리에 적합

**활용 예**
- 일/주/월 리포트
- 통계 분석, 추천 피처 생성

---

# Airflow란?
- 데이터 파이프라인을 관리하는 **워크플로우 오케스트레이션 도구**
- 작업을 언제, 어떤 순서로 실행할지 제어

**주요 기능**
- DAG 기반 작업 의존성 관리
- 스케줄링 및 재시도
- Spark, DB, ML 파이프라인과 연계

**역할**
- ETL 자동화
- 배치 작업 및 분석 파이프라인 운영

---

# Elasticsearch란?
- 대용량 데이터를 빠르게 검색·조회하기 위한 **분산형 검색 및 분석 엔진**
- JSON 기반 문서를 색인(Index)하여 **실시간에 가까운 조회 성능** 제공

**주요 특징**
- Full-text search 지원
- 분산 구조로 수평 확장 가능
- 스키마 유연한 문서(Document) 저장 방식

**주요 역할**
- 로그, 이벤트 데이터 검색
- 사용자 리포트 및 통계 조회
- 대시보드용 데이터 제공

**사용 시점**
- 빠른 검색 및 필터링이 필요한 경우
- 분석 결과를 사용자에게 실시간에 가깝게 제공해야 할 때

**일반 흐름**
- Data Source → Kafka/Spark → Elasticsearch → Dashboard / API
