## Docker 사전 학습 및 실습 정리

EC2 인스턴스로 Ubuntu 서버를 운영하기에 앞서, 연습 목적으로 로컬 환경에서 Ubuntu 실행 환경을 먼저 구성하여 실습을 진행하였다.

Oracle VirtualBox 하이퍼바이저 위에 Ubuntu Server(live-server 설치 이미지)를 설치하여 가상머신(VM)을 구성하였고, 해당 VM의 운영체제에 Docker를 설치하였다. 이를 통해 실제 클라우드 환경과 유사한 조건에서 Docker 기반 서비스 운영을 연습할 수 있었다.

---

### Docker의 Image와 Container 개념

Docker를 이해하기 위해 가장 먼저 이미지(Image)와 컨테이너(Container)의 개념을 정리하였다.

이미지(Image)는 애플리케이션 실행에 필요한 코드, 런타임, 라이브러리, 환경 설정 등을 포함한 **불변(immutable) 실행 템플릿**이다.
컨테이너(Container)는 해당 이미지를 기반으로 실제로 실행되는 **격리된 프로세스 단위의 인스턴스**이다.

즉, 특정 시점의 실행 환경이 정의된 이미지를 저장해 두었다가, 필요할 때마다 동일한 환경의 컨테이너를 생성하여 실행하는 방식으로 이해하였다. 이로 인해 개발 환경과 실행 환경의 일관성을 유지할 수 있다는 점이 인상적이었다.

---

### Docker 스토리지 개념

Docker를 조금 더 학습하면서 컨테이너의 데이터 관리 방식인 스토리지 개념도 함께 정리하였다. Docker의 스토리지 방식은 크게 다음 세 가지로 구분할 수 있다.

#### 1. Bind Mount

Bind mount는 Docker 호스트의 디렉토리와 컨테이너 내부의 디렉토리를 직접 연결하는 방식이다.
호스트 파일 시스템에 직접 접근하므로 개발 환경에서 자주 사용되며, 컨테이너와 무관하게 일반적인 파일 데이터를 관리할 때 적합하다. 다만 호스트 환경에 대한 의존성이 크다는 단점이 있다.

#### 2. Volume

Volume은 Docker가 직접 관리하는 독립적인 스토리지 공간으로, 컨테이너 생명주기와 분리된 데이터를 저장할 수 있다.
컨테이너를 삭제하거나 재생성하더라도 데이터가 유지되기 때문에, 데이터베이스와 같이 영속성이 필요한 데이터 관리에 적합하다고 이해하였다.

#### 3. tmpfs

tmpfs는 메모리(RAM)에만 존재하는 임시 저장소로, 컨테이너 종료 시 데이터가 즉시 소멸된다.
캐시 데이터나 민감하지만 영구 저장이 필요 없는 임시 데이터를 저장하는 용도로 활용할 수 있다.

### Docker 네트워크와 Nginx–Django 구성

Docker 네트워크는 컨테이너 간 통신을 위한 **가상 브리지 역할**을 한다.
같은 Docker 네트워크에 속한 컨테이너들은 IP 주소 대신 **컨테이너 이름을 사용해 서로 통신**할 수 있다는 점이 핵심이다.

실습에서는 Django 애플리케이션 컨테이너와 Nginx 컨테이너를 각각 띄우고, 두 컨테이너를 동일한 네트워크에 연결하여 리버스 프록시 구조를 구성하였다.

외부에서 `127.0.0.1:80`으로 들어온 HTTP 요청을 Nginx가 수신하고, upstream 설정에 따라 `8000` 포트에서 실행 중인 Django 서버로 요청을 프록시하는 방식이다. 이를 통해 웹 서버와 애플리케이션 서버의 역할 분리가 자연스럽게 이해되었다.

---

### EC2 보안 그룹과 포트 개방 이해

이 구조를 EC2 환경에 적용할 경우, 보안 그룹 설정에서 기본적으로 HTTP 서비스 제공을 위한 **80번 포트**를 개방해야 한다.
또한 원격 관리가 필요한 경우 **22번 SSH 포트**를 개방하되, 접근 가능한 IP를 제한하고 pem 키 관리를 철저히 해야 한다는 점도 함께 정리하였다. 이는 최소 권한 원칙에 기반한 설정으로 이해하였다.

### 데이터베이스 실습 및 기술 확장

실습 과정에서는 PostgreSQL을 로컬 설치 방식과 Docker 컨테이너 방식 두 가지로 구성해 보았다.
Django는 1학기 수업에서 이미 데이터베이스 연동을 경험했기 때문에 큰 어려움은 없었으나, 이번 프로젝트에서는 Java Spring Boot를 사용할 예정이므로 해당 환경에서의 DB 연동 방식은 추가 학습이 필요하다고 느꼈다.

### Docker Compose에 대한 이해

앞서 학습한 Docker의 개념들을 실제로 가장 편리하게 활용할 수 있게 해준 도구는 **docker-compose**였다.

`docker-compose.yml` 파일 하나로 여러 컨테이너의 서비스 구성, 네트워크, 볼륨, 의존성을 정의할 수 있으며, `up`과 `down` 명령어만으로 컨테이너의 일괄 실행과 중지가 가능하다는 점에서 매우 효율적이라고 느꼈다.

docker-compose는 단일 서버 환경에서 여러 컨테이너를 관리하기 위한 도구로, 개발 환경 및 소규모 배포 환경에서 특히 강력하다고 이해하였다.
